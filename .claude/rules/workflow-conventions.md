# Workflow Conventions

## Planning
- Skip plan mode for simple deliverables (single file changes, config updates)
- Use plan mode for multi-file implementation work
- When in doubt, ask

## Execution
- Read existing code before modifying it
- Match the existing patterns and style
- Keep changes minimal — only change what's needed
- Run tests before creating PRs

## Commits
- One logical change per commit
- Descriptive commit messages that explain why, not just what
- Don't commit secrets or sensitive configuration
